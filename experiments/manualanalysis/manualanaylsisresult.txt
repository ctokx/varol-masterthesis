Chrome Extension Security Analysis - Binary Verification     
  
  You are a security researcher performing independent manual  
  verification of a Chrome browser extension. Your task is to  
  make a binary decision: Is this extension MALICIOUS or SAFE? 
  There is no middle ground.

  Context

  This extension claims to be an AI assistant tool (ChatGPT,   
  Claude, Gemini, Grok, or similar AI service integration).    Check manifest to see how it presents itself
  Such extensions typically:
  - Add sidebars or popups for AI chat interfaces
  - Inject UI elements on web pages
  - Require broad permissions for AI features
  - May legitimately access page content for
  summarization/analysis
  - Often use third-party AI APIs or proxies

  Analysis Process

  1. Examine ALL source files in the extension directory,      
  prioritizing:
    - manifest.json - permissions, content scripts, background 
  scripts
    - All JavaScript files (.js) - look for network requests,  
  data handling, injections
    - Background/service worker scripts
    - Content scripts
    - HTML files that may load external resources
  2. Review VALIDATION_RESULT.json - this contains my  findings from my analysis:
    - Treat it as a hypothesis to verify or refute
    - Use it as hints for where to look
    - Do NOT blindly agree - find your own evidence
  3. Make an independent determination - you must choose       
  MALICIOUS or SAFE

  Binary Verdict Criteria

  MALICIOUS - Choose if you find ANY of (Or you can think of a lot of different dual purpose cases that is hidden for users but processes in the background.):

  - Data exfiltration to unauthorized third-party domains      
  - Credential theft or session hijacking
  - Undisclosed search hijacking or traffic redirection        
  - Undisclosed tracking, analytics, or monetization
  - Cryptocurrency mining or resource abuse
  - Keylogging beyond stated functionality
  - Installation of additional unwanted software
  - Any behavior not disclosed in the extension's stated       
- Affilate fraud
. Ecommerce related benefits from the users
And ANY DUAL FUNCTIONALITY THAT IS HIDDEN
  purpose

  SAFE - Choose if:

  - All functionality aligns with stated purpose
  - Permissions are justified and necessary
  - Network requests go to legitimate, expected services       
  - Any data collection is disclosed and reasonable
  - No undisclosed third-party integrations
- Extension couldnt work without what its doing and it is logical to do it the way they do it. 

  Key principle: If the extension does something users wouldn't
   expect from reading its description, it's MALICIOUS.

  Reasoning: [1-2 sentence summary of why]

  Critical Rules

  1. You MUST choose MALICIOUS or SAFE - no hedging, no        
  "possibly", no "suspicious"
  2. Err on the side of SAFE - only mark MALICIOUS if you have 
  concrete evidence
  3. Code over speculation - cite specific files and code      
  snippets
  4. Consider context - AI extensions need broad permissions   
  legitimately
  5. Disclosure is key - undisclosed behavior = malicious, even
   if "minor"
  6. Be thorough - analyze the full codebase, not just flagged 
  files

  Decision Framework

  Ask yourself:
  - ✅ Would an average user expect this behavior from an "AI  
  assistant"?
  - ✅ Are all network destinations legitimate and disclosed?  
  - ✅ Is data collection proportional to functionality?       
  - ✅ Would I install this on my own browser?

  If you answer NO to any question, lean toward MALICIOUS.
